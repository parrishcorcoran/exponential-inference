## Key Findings & Theory

### 1. The Binarization Gap (+1.18 nats)
- FP master weights from Stage 234 produce drift +0.054 under smooth per-group normalization (sphere forward).
- Binarized to `sign(W)·α_group` they produce drift +1.23.
- **Gap of +1.18 nats is the cost of replacing within-group magnitude variation with a single α.**
- This is information-theoretic, not a recipe failure. Mean-α IS already L2-optimal for 1-bit.
- To close it: need either (a) more bits per weight, (b) multi-axis multiplicative structure (factored α), or (c) richer encoding (codebook, residual).

### 2. Per-128 Grouping Beats Per-row α
- Stage 227 (per-row α, full W training): +1.70 at 1.013 bpw
- Stage 231 (per-128 random group, full W training): +1.40 at 1.125 bpw
- Per-128 has 8× more α resolution per row (for in=1024 Linears). The extra α capacity is worth the +0.11 bpw cost. Within-group variance halves vs per-row.

### 3. Magnitude Permutation Is a Small Win
- Stage 232 vs 231: -0.02 nats from sorting input dims by magnitude before per-128 grouping.
- Within-group variance reduced by ~sqrt(in_f / 128) vs random sequential grouping.
- Permutation overhead: ~0.01 bpw.

### 4. Carry-Flip Pair Structure Absent in Weights
- Test in `scripts/test_carry_flip_weight_structure.py` (results: `results/carry_flip_weight_structure.json`)
- Stage 77's 5+5 hologram pattern (phase-conjugate cluster pairs) found in token attribution but NOT in weight rows of Qwen3-0.6B.
- z-scores ~0 for anti-pair / orthogonal-pair counts at K=4..32 across 14 tested Linears.
- **Conclusion:** No exploitable sub-bit encoding via paired-row structure. The 5+5 pattern is attribution-specific.

### 5. Bake Cycle: Diminishing Returns at Per-128 Level
- Original Stage 226 bake (per-row sphere) drove FP weights from raw → +0.14 — huge structural reorganization.
- Stage 235 bake (per-128 sphere) on Stage 234 best → sphere forward improved from +0.29 to +0.054.
- But binary forward after Phase 1 is +1.63 (worse than Stage 234's +1.23) — the per-128 sphere structure doesn't directly translate to better binary.
- Phase 2 STE would have to recover this, but Mac couldn't run it (memory).

### 6. Mac Compute Ceiling (CRITICAL)
- **What fits at ~1 s/step:** 147M body W training only (Stages 227, 231, 232, 234, 235 P1)
- **What swaps to disk (30+ s/step):** 147M body W + ANY extra learnable params (α_col, α_row, factored α, etc.)
- **What works at low scope:** ~290K trainable (α-only, frozen W) — Stages 233 v2, 236 v2
- **The wall:** any time we add nn.Parameters BEYOND body W to the autograd graph, even tiny ones (287K), Mac swaps. Probably because the autograd graph topology with two parameter classes triggers different memory allocation.
- **What this blocks:** factored α + W retraining, hidden-state distillation, multi-bit experimentation.

## Architecture Reference

### Common Modules (across all stages)

```python
class PerRowSphereLinear(nn.Module):
    """Stage 226 architecture. Forward enforces row L2 = locked radius."""
    def forward(self, x):
        W_unit = self.weight / self.weight.norm(dim=-1, keepdim=True).clamp(min=1e-8)
        W_eff = W_unit * self.row_radius  # row_radius is buffer locked at init
        return F.linear(x, W_eff, self.bias)

class SubLNWrappedLinear(nn.Module):
    """BitDistill SubLN insertion before o_proj and down_proj inputs."""
    def forward(self, x):
        rms = x.float().pow(2).mean(dim=-1, keepdim=True).add(self.eps).sqrt().to(x.dtype)
        x_normed = self.subln_gain * x / rms  # subln_gain learnable
        return self.wrapped(x_normed)

class _BinarizeGroupSTE(torch.autograd.Function):
    """Forward: sign(W) * mean(|W|) per group. Backward: identity (STE)."""
    @staticmethod
    def forward(ctx, W, group_size):
        # ... reshape to groups, compute α=mean(|W|)/group, multiply sign
        return W_q
    @staticmethod
    def backward(ctx, grad_output):
        return grad_output, None  # straight-through

class BinaryGroupLinearWithPerm(nn.Module):
    """Per-128 binary + magnitude-sorted input permutation. Stages 232+."""
    def forward(self, x):
        x_perm = torch.index_select(x, -1, self.perm)
        W_q = binarize_group_ste(self.weight, self.group_size)
        return F.linear(x_perm, W_q, self.bias)
```

### Coverage Conventions
- `TARGET_NAMES = ("q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj")` — 7 Linear types per layer × 28 layers = 196 quantized Linears
- `BODY_TRAINABLE_SUFFIXES = ("o_proj", "down_proj")` — Stage 189 scope, Mac-feasible at 147M params
- SubLN inserted before o_proj and down_proj inputs only (BitDistill convention)

### Distillation Recipe (Stages 231+)
- Loss = α·CE + β·KL_topK_temperature
- α=1.0, β=1.0
- KL: top-K=64 tokens (vs full 151K vocab — denser signal, less tail noise)
- Temperature T=2.0 (softens both teacher and student before KL)
- T² scaling preserves gradient magnitude (Hinton convention)

### Cosine LR Schedule
- Standard: peak 2e-5 → 0 over training horizon (or 1e-5 if continuing from converged checkpoint)
- Final phase has LR effectively 0, signs commit, oscillation envelope collapses
