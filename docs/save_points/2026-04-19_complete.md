## I. The Core Discovery

Transformer hidden states live on a ~9-11 dimensional manifold inside a 1024-5120 dimensional ambient space. This is invariant across model size (0.6B–32B), architecture (dense, MoE, ternary/BitNet), and quantization. The manifold dimension appears to be a property of the tokenizer family, not the model.

**Implication:** current inference computes in 1000+ dimensions when only ~10 carry the decision. The architecture is inverted — wide where it should be narrow, uniform where it should be adaptive.

## II. The Physics Framework

### The Holographic Principle Applied to Transformers

- **Manifold (~10 dim) = the boundary.** It encodes the state — where the model is in its decision space.
- **MLP intermediate (3072-13824 dim) = the bulk.** It's the compute medium that materializes each layer's rotation of the state. NOT redundant — it's what the holographic reconstruction requires.
- **Each layer = a rotation/projection of the hologram.** More layers = more projections = higher resolution of the manifold.

### Boundary vs Bulk: The Unified Principle

**Every failed compression attacked bulk. Every successful compression attacked boundary.**

| Failed (bulk) | What it tried |
|---|---|
| Stage 6: SVD weight factoring | Compressing MLP weights |
| Stage 35: rank-k MLP factoring | Narrowing MLP intermediate |
| Stage 36: rotation-native narrow MLP | Same |
| Stage 42: intermediate-dim pruning | Dropping bulk dimensions |

| Succeeded (boundary) | What it compressed |
|---|---|
| Finding 04: head pruning | 80% heads prunable, 100% token match |
| Stage 38: KV cache compression | 8× KV reduction |
| Finding 09: early exit | Skip layers via stabilization_depth |

### Two Coordinates on the Manifold Surface

The manifold is a curved surface. Each token's position on it determines:

1. **Width** — how many attention heads this token needs. Single-mode distribution → few heads. Multi-modal → many heads. This IS MoE routing but physics-informed.

2. **Length** — how many layer-projections needed to resolve the token. Easy tokens resolve in fewer projections. Hard tokens need all layers.

**Depth (MLP intermediate) is NOT a coordinate.** It's fixed — the holographic projection mechanism. You can't reduce it.

### The Architecture

```
Token arrives
  → Router reads manifold position (from previous step's attention sharpness + features)
  → Router outputs: (N_heads, N_layers) for THIS token
  → Custom forward: activate N_heads, run N_layers, MLP full at every layer
  → Emit token
  → Feed sharpness signal to router for next token
```

This is MoE + early exit, unified by the manifold, with the bulk preserved.

## III. The 10 Findings

### Finding 01: Universal Manifold Dimension
- **9.07–10.89 intrinsic dimension** across 9 models (Qwen 0.6B–32B, BitNet 2B, Phi-2)
- Measured via TwoNN (Facco et al. 2017), validated on synthetic data
- Dimension appears tokenizer-bound, not architecture-bound

### Finding 02: Universal Rotation Curve
- Per-layer basis rotation follows the **same curve shape** when normalized to [0,1] depth
- Pearson r > 0.97 across all model pairs tested
- Shape: sharp drop at layer 0→1, rises to plateau 0.93-0.98

### Finding 03: Universal Phase Transition at Layer 0→1
- Every model's **largest basis rotation** is at the embedding→layer-1 boundary
- This is where the model transitions from token-embedding space to task-feature space

### Finding 04: 80-83% Head Pruning
- **80-83% of attention heads can be skipped per step with 100% token match**
- Signal: attention sharpness (1 - normalized entropy) from previous step
- Active heads ≈ manifold dim (2.5-4.8 heads × dims/head ≈ 8-10 dims)

### Finding 05: Manifold Floor
- Rank-k factored compression has a **size-independent parameter floor** (~80-160M)
- 0.6B at rank-32 = 20M params → BELOW floor → always fails
- 14B at rank-32 = 270M → ABOVE floor → Matryoshka nesting works (100%)
- BUT: Matryoshka nesting (factored-vs-factored) ≠ factored-vs-original (0% match)

### Finding 06: Four Entropy Descent Profiles
- Linear decline, bell curve, plateau, mid-generation spike
- Spikes = saddle crossing between basins (RSB hierarchy traversal)
- Only predicted by spin-glass physics, not by statistical learners

### Finding 07: Easy-Token Routing Signals
- Free signals (attention entropy, hidden norm, step size) correlate with difficulty
- Combined linear R² = 0.15-0.29 (weak individually, useful in combination)
- Binary routing viable (~70% correct)

### Finding 08: Minimal 8-Feature Set
- 8 orthogonal features capture 80% of routing power
- Each probes a different physics axis
- Cost: microseconds per step

### Finding 09: Logit-Lens Stabilization (Strongest Predictor)
- Stabilization_depth: r = +0.495 correlation with output entropy
- Adding 6 logit-lens features jumps LOPO R² from 0.293 → 0.448
- Validates the "layer-as-rotation" reframe

### Finding 10: Holographic Compressibility
- Names the technique: **Holographic Matryoshka**
- Boundary = compressible. Bulk = not compressible.
- Predicts all prior successes and failures retroactively

6. **Head masking**: 20/40 heads → coherent text ("be a lot more than just robots and self")

1. **Rank-k weight factorization**: 0% vs original on every attempt (Qwen 0.5B, 3B, 14B, Llama 8B)

3. **Output-only bottleneck on standard models**: Qwen 0.5B/3B plateau at ~43% (below manifold floor). BitNet Hydra was 97% but BitNet-specific
4. **Weight SVD**: 33× wall-clock but 0% quality (wrong basis)
5. **Static grid tests**: repeatedly shown to be unhelpful — the system must be dynamic

## V. The Architecture (Current Understanding)

### What It Is
A standard transformer where:
- **Width** (number of active heads) is dynamic per token, controlled by manifold position
- **Length** (number of layers) is dynamic per token, exit when prediction stabilizes
- **Depth** (MLP intermediate) is always full — the holographic projection medium
- **Router** reads manifold features (attention sharpness, hidden norm, entropy) and outputs (width, length) per token

### Why It's Better Than MoE
- MoE: routes to arbitrary experts, runs all layers, learned routing
- This: routes to specific heads, exits at the right layer, physics-informed routing
- MoE stumbled into "thin width, full length" without knowing why
- We know why: the manifold is low-dimensional, the holographic reconstruction needs depth

### Why It's Better Than CALM
- CALM: checks confidence at every layer (overhead exceeds savings)
- This: router decides ONCE based on manifold position (near-zero cost)
- CALM: binary exit (run or don't)
- This: continuous (width AND length vary smoothly)

### Why It's Better Than Speculative Decoding
- Spec decode: draft → verify → accept/reject (two models, verification overhead)
- This: route → compute at exact resolution → emit (one model, no verification)
- The router IS the verification — it knows the manifold, it doesn't need to check

### The Pin: Higher Head Count
- 40 heads gives 1/40th width granularity
- 128 heads (via distillation) would give 1/128th granularity
- More heads = finer routing = more efficient packing
- Distillation recipe: same hidden dim (5120), reshape attention to more thinner heads, KL distill against teacher

## VIII. What NOT To Do (Proven Dead Ends)

1. **Rank-k weight factorization** at any rank — 0% vs original, every time
2. **PCA inter-layer compression** — overfits to calibration data
3. **Weight SVD** — wrong basis (variance ≠ prediction)
4. **Compress MLP intermediate (bulk)** — Finding 10 says this can't work
5. **Static grid tests** — the system is dynamic, static measurements waste time
6. **Factored model as its own teacher** — trains to match garbage perfectly
7. **Checking confidence at every layer** — overhead exceeds savings
8. **Classifying tokens as "easy" or "hard"** — it's a continuum
