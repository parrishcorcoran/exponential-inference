## The project in one sentence

Transformer hidden states live on a low-dimensional (~10) curved
manifold that is a property of the tokenizer family. Every LLM is an
imperfect approximation to this manifold. Generation is a geodesic
traversal on its surface. Compute can be compressed on the boundary
but not in the bulk. A small model trained against the measured
manifold — not against a teacher's approximation — may exceed any
teacher of any size.

## Candidate Finding 12 (current active hypothesis, not yet formalized)

**The manifold is a curved landscape with topography, and generation is
a geodesic traversal with adaptive length.**

Evidence synthesis from existing findings:

- Finding 02's smooth rotation curve = universal connection on the manifold
- Finding 03's phase transition = entry from embedding into the bulk
- Finding 06's four entropy profiles = topographic shadow patterns of paths
- Finding 09's stabilization_depth = geodesic length per token
- Finding 11's RG + quantum measurement = annealing dynamics along geodesic

Testable architectural claims:

1. One KV head per layer sufficient (exact manifold point, no ensemble
   averaging needed)
2. Width (head count) per token = local manifold curvature
3. Geodesic integration time adaptive (not fixed `t_span=[0,1]`)
4. **Training target = measured manifold**, not teacher output

Claim 4 is the big one. If the manifold is measurable independently of
any teacher (via tokenizer + corpus geometry), training against the
measurement rather than a teacher's approximation should break the
teacher ceiling. A small model could exceed a large one by learning
the physical truth rather than imitating an approximation.

## IMPORTANT CORRECTIONS to prior claims

### 1. Holographic Matryoshka trained-weights approach is DEAD

The Strix commit `38a9967` said *"🏆 Qwen3-14B rank-32: 100% token
match at 160× compression"*. Closer reading of the eval code revealed
this compared student-at-k against student-at-k_max (both factored),
NOT student vs the original unfactored teacher. The
`validate_14b.log` file (committed alongside the run) shows the
factored 14B student's actual output on real prompts:

```
Prompt: 'The future of artificial intelligence will...'
  Original: be shaped by the convergence of multiple disciplines...
  Factored: PropertyParamsPropertyParamsPropertyParams...
```

Pure degenerate output on every prompt tested.

**Trained rank-k weight factorization does not reproduce the teacher,
at any tested size (0.5B, 3B, 8B, 14B).** This was fixed in commit
`48cdb65`.

### 2. "Holographic Matryoshka" now refers to a runtime architecture

Not a trained weight factoring. The technique is:
- Width per token: dynamic head masking (need better verification —
  see below)
- Length per token: early-exit at stabilization depth
- Depth preserved: MLP intermediate full (Finding 10)
- Matryoshka = nested inference widths (width=5 ⊂ width=40), NOT a
  trained rank ladder

### 3. Strix's dynamic routing claims need independent verification

The `manifold_inference.py` script as committed contained three bugs
that would prevent it from running as-is on Qwen3 (RoPE convention
mismatch, head-scale broadcast shape wrong, output reshape dim wrong).
When fixed and run on Qwen3-0.6B (stage 50), no configuration produced
coherent output — 0/40 match at every width/length combination.

Either:
- The approach is scale-dependent (works on 14B, not 0.6B)
- Strix's 14B "coherent" was marginal
- Strix was running different code than what was committed

Need a committed Strix log with actual teacher-vs-student samples on
14B before building on this claim.

### 4. The Geodesic claim has no validation log

Strix's commit `cfd5334` (2026-04-20) introduced a continuous-ODE
architecture replacing the transformer's discrete layer stack (files:
`train_geodesic*.py`, `model_package/modeling_exponential.py`). Claims:

- 2.4× faster (62.3 tok/s vs 26)
- 43× smaller (13.9M trainable params)
- "Matches 0.6B teacher"

**No committed validation log.** The claims exist only in the commit
body. Given Strix's pattern of overclaiming so far (see #1 above),
treat as architecturally interesting but **not verified** until a
validation log is produced.

## Gotchas a new Claude must know

1. **The user (Parrish Corcoran) is physics-first.** Don't revert to
   ML-conventional framings when surprised. Failures are narrowing
   measurements.

2. **"Match" means teacher-vs-student, always.** Self-consistency
   within a compressed model is not a quality claim. Grep the eval
   code if any commit claims "100% match."

3. **"Coherent" is not a number.** Either show token-match rate or
   show actual samples side-by-side.

4. **The user rejects hybrid framings.** When tempted to say "it's
   this half and that half," look harder for the unifying principle.

5. **Don't trust commit messages without checking the code.** Several
   Strix commits had inflated claims. Read the log/eval output.

6. **Bulk compression always fails.** Don't propose new d_int
   compression schemes. Finding 10 is solid.

7. **Manifold floor (~80-160M) is real.** 0.6B failures are often
   below-floor artifacts, not falsifications of the approach.

8. **Kernel hooks are expensive on MPS, cheap on ROCm.** Prefer ROCm
   (Strix) for production speedup tests.

9. **KV is a boundary object, separable from heads/width/length.**

10. **Surviving physics frames: RG flow (Finding 11 Frame 3), quantum
    measurement / pointer selection (Frame 6a). Both confirmed.
    Don't re-propose fractal, CFT scaling, RSB glass, parallel
    transport — all falsified (stages 45-48).**

## The open frontier question

**Can we measure the manifold independently of any teacher, and train
against that measurement to break the teacher ceiling?**

Everything else is in service of this question. If the answer is yes,
a 0.6B model trained correctly against the measured manifold could
exceed a 400B model trained conventionally. That's the project's
ultimate claim.

If the answer is no — if every "manifold measurement" turns out to be
contaminated by the measurement apparatus (usually an existing LLM) —
then Holographic Matryoshka settles into a runtime-routing
architecture with measurable but modest speedups, not a paradigm shift.

The geodesic architecture (Strix's recent work) is a vehicle for
testing this. The test needs (a) a clean manifold measurement protocol
and (b) a committed validation log comparing the geodesic's quality
against any teacher used in the measurement.
